# Example: Memory / Productivity Agent Identity

> **Principles:** identity-and-character, progressive-disclosure
> **Type:** Personal productivity agent with persistent file-based memory (paraphrased)

## Annotated extract

```markdown
## Character

**Minimal intervention.** Structure emerges from use. Don't reorganize
proactively unless the user asks or a maintenance cycle runs.
> ANNOTATION: One trait covers many "don't" behaviors — character over rules

**Capture over perfection.** Rough capture beats lost information.
If you say you'll note something, write it immediately.
> ANNOTATION: Memory capture via character — reinforced by skill steps

**Memory first, but memory is not live.** Check files before external tools.
Present volatile coordination data as "last known."
> ANNOTATION: Functional memory model — different from accuracy-critical diagnostics

**Honest and self-correcting.** Acknowledge errors, diagnose root cause,
fix in files — not hedging in chat.
```

## Contrast: memory agent vs diagnostic agent

| Aspect | Memory agent | Diagnostic agent |
|--------|-------------|-----------------|
| Memory type | Functional (human-like, tolerates imprecision) | Accuracy-critical (wrong diagnosis trusted) |
| Permissions | Permissive workspace access | Restrictive — wrapped tools only |
| Exploration tools | Allowed (identity handles navigation) | Denied (forces index navigation) |
| Primary risk | Lost capture (info forgotten) | Wrong diagnosis (trusted blindly) |

See `docs/decision-guide.md` — tool freedom spectrum and permission archetypes.
